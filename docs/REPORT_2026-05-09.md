## 3. Phase 5 — Realtime Chat & Web-bridge Auth

### 3.1 Centrifugo client mobile (`src/lib/realtime.ts`)

- `connectRealtime()` — single client per app, refcounted: subscribe pertama
  trigger connect, unsubscribe terakhir tidak disconnect (idle timeout 60s
  untuk reconnect cepat saat user buka chat lain).
- `subscribe<T>(channel, handler)` → unsubscribe function.
- Auth: short-lived JWT dari `/api/centrifugo/token` (Bearer) — TTL 5 menit,
  auto-refresh sebelum expire.
- **Graceful fallback**: kalau WS unreachable (mis. nginx belum proxy
  `/connection/websocket` di prod), client return `null` → caller fallback
  polling 5s.

### 3.2 ChatThread integration (`app/pesan/[id].tsx`)

```ts
useEffect(() => {
  void load();
  let unsub = null, polling = null;
  void (async () => {
    const client = await connectRealtime().catch(() => null);
    if (client) {
      unsub = client.subscribe(directThreadChannel(id), () => void load(true));
    } else {
      polling = setInterval(() => void load(true), 5000);
    }
  })();
  return () => { unsub?.(); polling && clearInterval(polling); };
}, [id, load]);
```

Trade-off: tidak append manual saat realtime event datang. Refetch full thread
supaya tidak perlu dedup logic. Network cost minimal (~3KB per fetch, 1x per
message).

### 3.3 Web-bridge endpoint

**Problem**: Mobile pakai Bearer JWT, tapi WebView (live session room,
tutor analitik, sertifikat PDF) butuh **cookie session** karena Next.js
middleware + RSC pakai cookie. Pass Bearer header ke WebView tidak portable
(Android `WebView.loadUrl(url, headers)` tidak persist headers di subsequent
nav, iOS `WKWebView` butuh extra setup).

**Solution**:

```
POST /api/mobile/web-bridge        Bearer JWT
  → { code: "abc123" }              one-time code, Redis TTL 60s

GET /m/x/[code]?next=/live-session/123
  → set-cookie senopati_mobile_token=<jwt>; HttpOnly; Secure; SameSite=Lax
  → 302 redirect to ?next param
```

Cookie disisipkan ke 3-source `getCurrentUser`:

```ts
// src/lib/session.ts
export async function getCurrentUser(): Promise<SessionUser | null> {
  // 1. Bearer header (mobile native API)
  const bearer = await tryBearer();
  if (bearer) return bearer;
  // 2. mobile cookie (WebView yang spawn dari mobile)
  const cookie = await tryMobileCookie();
  if (cookie) return cookie;
  // 3. NextAuth session (web browser)
  return await tryNextAuthSession();
}
```

Mobile helper `openAuthenticatedWebView(path)`:

```ts
export async function openAuthenticatedWebView(path: string) {
  const { code } = await apiFetch<{ code: string }>("/api/mobile/web-bridge", {
    method: "POST",
  });
  const url = `${API_BASE}/m/x/${code}?next=${encodeURIComponent(path)}`;
  await WebBrowser.openBrowserAsync(url);
}
```

Dipakai untuk: live session room, sertifikat PDF, analitik tutor, materi
upload, dan join external Zoom link.

## 8. Phase 10 — Offline Cache Layer

### 8.1 Cache module (`src/lib/cache.ts`)

```ts
type CacheEntry<T> = { data: T; cachedAt: number };

readCache<T>(key: string): Promise<CacheEntry<T> | null>
writeCache<T>(key: string, data: T): Promise<void>
clearCache(): Promise<void>
invalidateCache(patterns: string[]): Promise<number>  // wildcard "*" supported
getCacheStats(): Promise<{ keys: number; bytes: number }>
```

Storage: AsyncStorage prefix `cache:` per key. Wildcard match: `*` cocokin
suffix apa pun (mis. `/api/student/modules*` invalidate semua varian query).

### 8.2 useFetch enhancement (`src/lib/useFetch.ts`)

Stale-while-revalidate:

1. Mount → `readCache(path)` immediate. Kalau ada → `data = cached.data, loading = false, fromCache = true`.
2. Paralel → fetch fresh. Sukses → `writeCache + setData`.
3. Network error → keep cached data, set `error` (UI tampilkan banner offline).

Tambahan dep `reconnectTick` dari `useNetwork()` — saat tick naik (offline → online), trigger refetch otomatis tanpa user action.

### 8.3 Cache invalidation di apiFetch

```ts
apiFetch(path, { method: "POST", body, invalidate: ["/api/messages/threads", "/api/messages/threads/123"] })
```

`invalidate` di-jalankan setelah response sukses. Wildcard supported. Tidak
fail kalau cache miss.

Dipakai konsisten di mutation jalur:
- POST chat → invalidate thread + thread list
- RSVP → invalidate live event + live list
- Profile save → invalidate profile + me
- Mark notif read → invalidate notif list

### 8.4 OfflineBanner

`<OfflineBanner />` di-mount di root. Visible saat `!isOnline`. Tone amber,
text "Mode offline — beberapa konten ditampilkan dari cache".
`<GlobalOfflineBanner />` versi sticky bottom dengan animasi slide-in.

## 10. Phase 12 — Offline Action Queue

### 10.1 Action queue (`src/lib/action-queue.ts`)

```ts
type QueuedAction = {
  id: string;             // {timestamp}-{random}
  method: "POST" | "PATCH" | "PUT" | "DELETE";
  path: string;
  body?: unknown;
  invalidate?: string[];  // di-passed ke apiFetch saat flush
  label?: string;         // human readable: "Pesan ke Andi"
  attempts: number;
  createdAt: number;
};

enqueueAction(action) → QueuedAction
flushQueue() → { flushed, failed, remaining }
getPendingCount() → number
clearQueue() → void
```

Storage: AsyncStorage key `action_queue_v1`.

**Retry policy**:
- Sukses → drop
- 401/403 → drop (token expired, gak akan recover)
- Network/5xx → keep, attempts++
- attempts >= 3 → drop (give up)

**Concurrency**: module-scope `flushing` boolean lock supaya `flushQueue`
tidak race kalau dipanggil paralel (NetInfo reconnect callback + manual button).

### 10.2 Wiring offline send

`app/pesan/[id].tsx` `onSend()`:

```ts
if (!isOnline) {
  await enqueueAction({
    method: "POST",
    path: `/api/messages/threads/${id}`,
    body: { body: text },
    invalidate: [`/api/messages/threads/${id}`, "/api/messages/threads"],
    label: `Pesan ke ${data?.otherUser.name ?? "—"}`,
  });
  setDraft("");
  setPendingHint("Pesan akan dikirim saat online.");
  return;
}
// Normal send path... + ApiError network fallback juga enqueue
```

Pending hint banner amber tampil 3 detik antara message list dan input bar.

### 10.3 Auto-flush saat reconnect

`network.tsx` `NetworkProvider`:

```ts
if (wasOffline) {

}
```

### 10.4 UI manual flush di Profil

Below cache row, conditional `pending > 0`: amber row icon clock + "{N} aksi
menunggu dikirim saat online" + tombol "Sinkron". Tap → `flushQueue()` →
