## 2. Bug fixes beta test (smoke.test@example.com)

### 2.1 Bug #1 — modul yang sudah diakhiri tutor tidak tertanda selesai di `/program/paham-ai`

**Akar masalah**: 2 jalur completion modul (final-exam quiz dan live-session
end), tapi `getPahamAIProgress` hanya cek jalur final-exam. Hampir semua
siswa beta lulus via live-session → `completionPct=0` di program tracking.

**Fix**:
- `src/lib/progress-server.ts:hasPassedFinalExam` — terima 2 jalur
- `src/lib/progress-server.ts:getPahamAIProgress` — same fix di komponen check

### 2.2 Bug #2 — hasil belajar siswa tidak masuk ke dashboard tutor + admin

**Akar masalah**: Recompute scorecard + psych profile cuma jalan di cron weekly
Sunday. Beta test 8/5, cron next 10/5 → data lag 2 hari.

**Fix**: Event-driven recompute via lib baru `recompute-trigger.ts`:
- `triggerStudentRecompute(studentId)` — fire-and-forget Promise.allSettled
- `triggerStudentLearningComplete(studentId, moduleSlug?)` — dipanggil saat:
  - Live event end (`applyLiveSessionProgress`)
  - Narrative complete (`/api/cerita/result/route.ts`)
- `issueModuleCertificateIfCompleted` — auto-issue cert kalau ModuleProgress.completedAt set

Manual recompute dijalankan untuk smoke.test → 4 cert SENO-XXXX, scorecard
55.18 berkembang, psych profile 1 entry period 2026-05.

### 2.3 Bug #3 — admin penilaian siswa tidak menampilkan data smoke.test

**Akar masalah**: `getMonthlyPeriod()` default ke **bulan kalender lalu**
(April), padahal recompute event-driven nyimpan ke **bulan ini** (Mei). Page
admin query `periodStart=2026-04-01`, scorecard adanya di `2026-05-01` →
tidak ketemu.

**Fix**: `src/lib/tutor-scorecard.ts:getMonthlyPeriod` default ganti ke bulan
ini. Cron `quarterly-recompute` pass period eksplisit jadi tidak terdampak.

### 2.4 Bug #4 — konsumer Kafka mati, behavioral_events kosong

**Akar masalah**: `/src/workers/behavioral-consumer.ts` ada tapi **tidak ada
systemd unit**. Topic Redpanda `behavioral-events` punya 65 event lag — semua
event Jeda smoke.test (`narrative_beat_enter`, `narrative_choice`, `rsvp`)
mandeg di Kafka, tidak masuk DB.

**Fix**:
- Bikin `senopati-behavioral-consumer.service` (systemd user)
- Konsumer drain 65 event lag → 0 dalam <5 detik
- Smoke.test psych profile lengkap: `beatEnters=27`, `narrativeChoicesMade=13`,
  `completionRate=48.1%`, `rsvp=4`

### 2.5 Field misnomer

`retention.avgBeatTimeMs` → `avgChoiceDwellMs`. Sebelumnya nama mengindikasikan
"time per beat" tapi semantic-nya rata-rata dwell time saat siswa pilih opsi
(narrative choice) atau jawab quiz. Rename di 3 lokasi (psych-profile-language,
recompute-psych, UI label).

## 6. Profile Completion Gate

### 6.1 Mekanisme

- **JWT carries flag** — `token.profileComplete: boolean` ditambah di NextAuth
  jwt callback. Helper `deriveProfileComplete(uid, role)` query DB:
  tutor/admin → `true`, student → `true` kalau `profileCompletePct >= 100`.

- **Middleware guard** — kalau `role === "student" && profileComplete === false`,
  redirect ke `/siswa/profil?first=1` kecuali path whitelist (`/siswa/profil`,
  `/akun`, `/logout`).

- **First-time hero** — tone amber dengan eyebrow "Selamat datang!" dan title
  "Isi biodata dulu, baru mulai belajar".

- **Form session refresh** — `StudentProfileForm` panggil `useSession().update()`
  setelah save mencapai 100%, lalu `router.replace("/dashboard")`.

provinceId, schoolName, schoolGrade, educationTrack, parentName, parentPhone`.

StudentProfile, gate skip otomatis.
